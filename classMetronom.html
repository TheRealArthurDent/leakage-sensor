Parameters
cycleDurationThe duration of one cycle of the loop.
skipMissedCycles

In case a cycle has been missed - due to stuff happening within the loop taking more time than the defined cycle duration - this bool defines whether the missed cycle should be skipped or not. In this case true will make metronom's waitForNextCycle just wait until the next regular cycle, thereby skipping at least one cycle, whereas false will make metronom's waitForNextCycle return immediately and thereby trigger the next iteration of the loop in order to make up the delay.

Warning
Using skipMissedCycles = false in conjunction with a loop that usually takes more time than cycleDuration will lead to a growing delay that can never be made up.
